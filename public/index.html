Hype5 Server

Built and open-source project API which serves an aggragate of data from music blogs. The API operate via compiling all requested data via a web crawling and returns response afater web crawling completes. Project runs in Docker container.

Note: Given that this project is based on PhantomJS and is crawling web address that use javascript responses will take a WHILE (avg on good connection 3-7sec #firstworldproblems). Therefore, design accordingly and use loaders and animation in places where the data is being requested.

Valid Endpoint :

/hypeor /hype/top - GET: Will retrieve the top most popular tracks ranked this week as of today

/hype/remixes - GET: Will retrieve the top remixes only most popular tracks ranked this week as of today

/hype/noremixes - GET: Will retrieve the top original un-remixed tracks only most popular tracks ranked this week as of today

Parameters:

type ("popular" | "latest")

type="popular" (e.g.: /hype/noremixes?type="popular") - This is the default behavior and will grab the most popular remixes

type="latest" (e.g.: /hype/noremixes?type="latest") - Will retrieve latest non-remixes sorted by rank
